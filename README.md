## Uso

1. Clona el repositorio.
2. Instala node para Angular con `npm install @angular/cli`
3. Instala las dependencias con `npm install`.
    1. @angular/animations
    2. @angular/common
    3. @angular/compiler
    4. @angular/core
    5. @angular/forms
    6. @angular/material
    7. @angular/platform-browser
    8. @angular/platform-browser-dynamic
    9. @angular/router
    10. @ng-bootstrap/ng-bootstrap
    11. bootstrap
    12. font-awesome
    13. jquery
    14. ngx-bootstrap
    15. ngx-cookie-service
    16. ngx-socket-io
    17. ngx-toastr
    18. rxjs
    19. socket.io-client
    20. tslib
    21. zone.js
3. Ejecuta la aplicación con `ng serve`.
    1. Si quieres acceder desde otros dispositivos, puedes ejecutarlo con `ng serve --host 0.0.0.0` 
4. Abre tu navegador y visita `http://localhost:4200`.

Escribir un buen README es esencial para cualquier proyecto, ya que proporciona información valiosa a los usuarios y colaboradores potenciales. Aquí tienes algunas pautas para escribir un README efectivo:

### Estructura básica:

1. **Título y Descripción Breve:**
   - Proporciona un título claro y conciso que describa el propósito principal del proyecto.
   - Añade una breve descripción que resuma de qué se trata el proyecto.

2. **Estado del Proyecto:**
   - Indica claramente si el proyecto está en desarrollo, en versión beta, estable o cualquier otro estado relevante.

3. **Tabla de Contenidos:**
   - Si tu README es extenso, considera agregar una tabla de contenidos para facilitar la navegación.

### Instrucciones de Uso:

4. **Instalación:**
   - Proporciona instrucciones claras y concisas sobre cómo instalar tu proyecto.
   - Incluye requisitos previos, dependencias y comandos de instalación.

5. **Configuración:**
   - Explica cómo configurar y personalizar el proyecto.
   - Proporciona ejemplos de archivos de configuración si es necesario.

### Guía de Usuario:

6. **Uso:**
   - Proporciona ejemplos prácticos y explicaciones paso a paso sobre cómo usar tu proyecto.
   - Incluye ejemplos de código y comandos.

7. **Ejemplos:**
   - Agrega ejemplos de casos de uso típicos para mostrar cómo funciona tu proyecto en situaciones del mundo real.

### Contribuciones y Desarrollo:

8. **Contribuciones:**
   - Indica si estás abierto a contribuciones y cómo otros pueden contribuir.
   - Incluye pautas para la presentación de problemas y solicitudes de extracción.

9. **Desarrollo Local:**
   - Proporciona instrucciones para configurar un entorno de desarrollo local si es necesario.

### Documentación Adicional:

10. **Documentación:**
    - Enlaza a cualquier documentación adicional, como wikis o documentos más extensos.

11. **Licencia:**
    - Especifica claramente la licencia bajo la cual se distribuye el proyecto.

### Estilo y Formato:

12. **Formato y Estilo de Código:**
    - Si es aplicable, especifica las convenciones de estilo de código que deban seguir los colaboradores.

13. **Enlaces y Recursos:**
    - Agrega enlaces a la página principal del proyecto, la documentación, la comunidad y cualquier otro recurso relevante.

### Contacto y Soporte:

14. **Contacto:**
    - Proporciona información de contacto para preguntas, problemas o colaboraciones.

15. **Soporte:**
    - Indica cómo los usuarios pueden obtener soporte técnico, ya sea a través de problemas en GitHub, un foro, o cualquier otro medio.

### Actualizaciones y Historial de Cambios:

16. **Historial de Cambios:**
    - Lleva un registro de las versiones anteriores y los cambios realizados en cada versión.

### Consejos Adicionales:

- **Mantenlo Conciso:** Mantén el README claro y conciso. Los usuarios deben poder entender rápidamente qué hace tu proyecto y cómo pueden usarlo.

- **Usa Ejemplos Prácticos:** Proporciona ejemplos prácticos y casos de uso para ayudar a los usuarios a comprender cómo tu proyecto puede beneficiarlos.

- **Manténlo Actualizado:** Actualiza regularmente el README para reflejar cambios importantes en el proyecto.

Recuerda que un buen README puede marcar la diferencia en términos de adopción y colaboración en tu proyecto.




¡Claro! Los archivos con extensión `.md` son archivos Markdown, un formato de marcado ligero que se convierte fácilmente a HTML. Aquí tienes algunos elementos básicos de formato en Markdown:

### Encabezados:

```markdown
# Encabezado 1
## Encabezado 2
### Encabezado 3
#### Encabezado 4
##### Encabezado 5
###### Encabezado 6
```

### Texto en Negrita y Cursiva:

```markdown
**Texto en negrita**
*Texto en cursiva*
```

### Listas:

Listas no ordenadas:

```markdown
- Elemento 1
- Elemento 2
  - Subelemento A
  - Subelemento B
```

Listas ordenadas:

```markdown
1. Elemento 1
2. Elemento 2
   1. Subelemento A
   2. Subelemento B
```

### Enlaces:

```markdown
[Texto del enlace](URL)
```

### Imágenes:

```markdown
![Texto alternativo](URL de la imagen)
```

### Citas:

```markdown
> Esto es una cita.
```

### Líneas Horizontales:

```markdown
---
```

### Código en línea y Bloques de Código:

Código en línea: \`código\`

Bloque de código:

\```python
def ejemplo():
    print("Hola, mundo!")
\```

### Tablas:

```markdown
| Encabezado 1 | Encabezado 2 |
| ------------ | ------------ |
| Celda 1,1    | Celda 1,2    |
| Celda 2,1    | Celda 2,2    |
```

### Anidamiento:

Puedes combinar diferentes elementos de formato:

```markdown
**Texto en negrita _con cursiva_ y enlace [aquí](URL).**
```

Estos son solo algunos ejemplos básicos. Markdown es muy versátil, y hay muchas otras características que puedes explorar para dar formato a tus documentos de manera efectiva.
